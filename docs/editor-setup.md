# Editor Setup

ROCm-Oxide ships editor defaults for VS Code and rust-analyzer in the source
checkout and in projects generated by `cargo rocm-oxide new`.

## What is configured

The checked-in `.vscode/settings.json` files tell rust-analyzer to index both
the host crate and the `device-spike/` device crate. Generated projects use:

```json
{
  "rust-analyzer.linkedProjects": [
    "Cargo.toml",
    "device-spike/Cargo.toml"
  ]
}
```

The source workspace links the root crate, `device-spike/`, the device API
crate, proc-macro crate, and cargo/build tools.

## Device-code completion

Open `device-spike/src/lib.rs` and import the prelude:

```rust
use rocm_oxide_device::prelude::*;
use rocm_oxide_kernel::{kernel, kernel_contract};
```

Autocomplete should then discover the intended device-code path:

- `for_each_element(n, |i| { ... })`
- `i.as_usize()` and `i.is_in_bounds(len)`
- `input.read(i)`
- `out.set(i, value)`
- `math::sqrt_f32(...)`, `this_thread_block()`, atomics, vectors, and barriers

VS Code also gets snippets:

- `rocm-kernel-1d`
- `rocm-vector-add`

## Why check-on-save is disabled

The device crate is not a normal host-target Rust crate. A plain command such
as:

```sh
cargo check --manifest-path device-spike/Cargo.toml
```

checks for the host target and fails before reaching user code because AMDGPU
intrinsics are only available for `amdgcn-amd-amdhsa`. ROCm-Oxide builds device
code with nightly Rust, `-Z build-std=core`, and `-C target-cpu=<gfx arch>`.

For that reason, the project settings disable rust-analyzer's automatic
check-on-save and provide explicit tasks instead.

The settings also disable build-script execution inside rust-analyzer. That
keeps the editor from unexpectedly running the GPU build pipeline while you are
typing. Generated host bindings from `include!(env!("ROCM_OXIDE_DEVICE_BINDINGS"))`
are validated by `cargo build` / `cargo run`, not by editor startup.

## VS Code tasks

Generated projects include:

- `ROCm-Oxide: check scaffold`
- `ROCm-Oxide: doctor`
- `ROCm-Oxide: build host + device`
- `ROCm-Oxide: run app`

The source checkout includes:

- `ROCm-Oxide: doctor`
- `ROCm-Oxide: build device code`
- `ROCm-Oxide: hello Rust GPU`
- `ROCm-Oxide: offline verification`

Use those tasks, or run the same commands in a terminal, for real validation.

## Other editors

For editors that expose rust-analyzer settings directly, add the host and
device manifests to `linkedProjects` and disable check-on-save for the device
crate unless you provide a custom command that invokes the ROCm-Oxide build
pipeline.

Generated projects:

```json
{
  "linkedProjects": [
    "Cargo.toml",
    "device-spike/Cargo.toml"
  ],
  "checkOnSave": false
}
```

Source checkout:

```json
{
  "linkedProjects": [
    "Cargo.toml",
    "device-spike/Cargo.toml",
    "crates/rocm-oxide-device/Cargo.toml",
    "crates/rocm-oxide-kernel/Cargo.toml",
    "tools/rocm-oxide-build/Cargo.toml",
    "tools/cargo-rocm-oxide/Cargo.toml"
  ],
  "checkOnSave": false
}
```

Then validate with:

```sh
cargo run
```

inside a generated project, or:

```sh
cargo run --features device-spike --example hello_gpu_rust
```

inside the ROCm-Oxide source checkout.
