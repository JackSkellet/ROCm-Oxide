# Getting Started with ROCm Oxide

This guide walks you from a bare machine to a running GPU kernel written in Rust,
using the ROCm Oxide SDK.

---

## Prerequisites

### Hardware

| Requirement | Detail |
|-------------|--------|
| AMD GPU | RDNA 2 (gfx1030+), RDNA 3 (gfx1100+), RDNA 4 (gfx1201+), or CDNA 2/3 |
| Driver | AMDGPU 6.7+ (ships with ROCm 6.x) |

Run `rocminfo | grep gfx` to confirm your GPU is visible and note the architecture
string (e.g. `gfx1100`). You need this string when debugging build failures.

### Software

| Tool | Minimum version | Purpose |
|------|----------------|---------|
| ROCm | 6.0 (7.x preferred) | HIP runtime, HIPRTC, COMGR |
| Rust nightly | any recent nightly | `core` for `amdgcn-amd-amdhsa` target |
| `rust-src` component | — | Required by `-Z build-std=core` |
| `llc` (ROCm) | — | Lower LLVM IR → object file |
| `clang` (ROCm) | — | Link objects → `.hsaco` code object |
| `llvm-readelf` (ROCm) | — | Validate generated `.hsaco` |
| `llvm-objdump` (ROCm) | — | Optional: disassemble kernels |

ROCm tools are usually installed at `/opt/rocm/bin/`. Confirm they are on `PATH`:

```sh
which llc clang llvm-readelf llvm-objdump rocminfo
```

### Install the nightly toolchain and required components

```sh
rustup toolchain install nightly
rustup component add rust-src --toolchain nightly
```

The workspace `rust-toolchain.toml` pins the exact nightly version used for this
project, so `cargo` picks it up automatically inside the workspace.

## Quick health check

Run the built-in doctor command to verify your environment before writing any code:

```sh
cargo rocm-oxide doctor
```

The doctor checks for all required ROCm tools, the HIP runtime library, HIPRTC,
COMGR, at least one visible GPU, and a valid Rust nightly + `rust-src`. Fix any
items marked `✗` before continuing.

## Create your first project

```sh
cargo rocm-oxide new my-gpu-project
cd my-gpu-project
```

This creates:

```
my-gpu-project/
├── Cargo.toml             # Host crate (depends on rocm-oxide)
├── build.rs               # Runs rocm-oxide-build, copies HSACO + bindings
├── src/
│   └── main.rs            # Host entry point
└── device-spike/
    ├── Cargo.toml         # Device crate (no_std, amdgcn-amd-amdhsa)
    └── src/
        └── lib.rs         # Your GPU kernels live here
```

### What the generated code does

**`device-spike/src/lib.rs`** — A starter kernel `fill_indices`:

```rust
use rocm_oxide_kernel::kernel;
use rocm_oxide_device as gpu;

#[kernel]
pub unsafe extern "C" fn fill_indices(out: gpu::DeviceSliceMut<u32>, n: usize) {
    let i = gpu::global_id_x();
    if i < n {
        unsafe { out.write_unchecked(i, i as u32) };
    }
}
```

Every thread writes its own global index into the output buffer. This is the
simplest possible kernel that demonstrates the full Rust GPU pipeline.

**`src/main.rs`** — Loads the compiled kernel and launches it:

```rust
fn main() -> rocm_oxide::Result<()> {
    let device = rocm_oxide::Device::first()?;

    // Generated by rocm-oxide-build; exposes typed launch methods
    let kernels = DeviceKernels::load_embedded(&device)?;

    let n: usize = 1024;
    let mut out = rocm_oxide::DeviceBuffer::<u32>::new_zeroed(n)?;

    let config = rocm_oxide::LaunchConfig::for_num_elems(n);
    kernels.fill_indices(config, &mut out, n)?;

    let result = out.copy_to_vec()?;
    assert_eq!(result[42], 42);
    println!("GPU kernel ran successfully.");
    Ok(())
}
```

## Build and run

```sh
cargo run
```

On first build this triggers the full pipeline:

1. `build.rs` → invokes `rocm-oxide-build` as a subprocess.
2. `rocm-oxide-build` compiles `device-spike` for `amdgcn-amd-amdhsa` with
   `-Z build-std=core`, rewrites LLVM IR, runs `llc`, then `clang`, and
   produces `device_spike.hsaco`.
3. `rocm-oxide-build` emits `bindings.rs` (typed `DeviceKernels` struct) and
   `metadata.json` (argument type records).
4. `build.rs` copies the artifacts to `OUT_DIR` and sets env vars.
5. The host crate includes `bindings.rs` via `include!(env!("ROCM_OXIDE_DEVICE_BINDINGS"))`.
6. The host binary launches the kernel.

Subsequent builds only recompile when device-spike source changes.

## Verify the build

After a successful `cargo run`, check artifact integrity:

```sh
cargo rocm-oxide verify --quick
```

For a more thorough check including launch validation on the actual device:

```sh
cargo rocm-oxide verify
```

## Project layout conventions

| Path | Role |
|------|------|
| `device-spike/src/lib.rs` | GPU kernels annotated with `#[kernel]` |
| `src/main.rs` | Host code: open device, load kernels, allocate buffers, launch |
| `build.rs` | Orchestrates `rocm-oxide-build`; do not edit manually |
| `Cargo.toml` (workspace root) | Host crate; add host dependencies here |
| `device-spike/Cargo.toml` | Device crate; add device-side `no_std` dependencies here |

## Common errors

### `error: no HIP-capable device detected`

Check that the GPU driver is loaded (`lsmod | grep amdgpu`) and that your user has
access to `/dev/kfd` and `/dev/dri/renderD*`. You may need to be in the `render`
and `video` groups.

### `could not compile device crate: error[E0463]: can't find crate for 'core'`

The `rust-src` component is missing for the active nightly:

```sh
rustup component add rust-src --toolchain nightly
```

### `llc: command not found` / `clang: command not found`

ROCm tools are not on `PATH`. Add `/opt/rocm/bin` to your shell profile:

```sh
# Fish shell
fish_add_path /opt/rocm/bin
```

Then re-run `cargo rocm-oxide doctor` to confirm.

### `Architecture mismatch: metadata says gfx1100 but device is gfx1201`

The cached HSACO was compiled for a different GPU. Run `cargo clean` and rebuild:

```sh
cargo clean
cargo run
```

### `ROCM_OXIDE_DEVICE_HSACO not set`

`build.rs` did not run (or failed silently). Check `cargo build` output for
messages from the build script. Run with verbose output:

```sh
cargo build -v 2>&1 | grep rocm-oxide-build
```

## Next steps

- **Add a kernel**: Open `device-spike/src/lib.rs`, add a new `#[kernel]` function,
  rebuild. `DeviceKernels` automatically gains a typed method with the same name.
- **Use device types**: `#[device_global]`, `#[shared]`, `#[constant]` attributes
  from `rocm-oxide-kernel` let you declare GPU-side globals and LDS memory.
- **Read the API overview**: [`docs/api_overview.md`](api_overview.md) describes
  every public module and type in the SDK.
- **Explore examples**: The `examples/` directory contains end-to-end programs
  covering vector math, 3D rendering, atomics, and more.
- **Understand the build pipeline**: [`docs/toolchain-discovery.md`](toolchain-discovery.md)
  explains how `rocm-oxide-build` locates ROCm tools and resolves version
  mismatches.
